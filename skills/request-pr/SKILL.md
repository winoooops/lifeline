# /lifeline:request-pr — Open a PR end-to-end

Stages: branch sanity → resolve authoritative base → preflight → safe push → autogen title+body → create PR. Pairs with `/lifeline:approve-pr` (which finishes a PR).

## Invocation

```
/lifeline:request-pr               # current branch's commits, prompt to confirm
/lifeline:request-pr Y             # current branch's commits, auto-confirm (apt-style)
```

`Y` / `y` skips the confirmation prompt at Step 5 only. The pre-flight guards (detached HEAD, dirty tree, behind-or-diverged) are non-overridable.

## Key invariants

1. **Detached HEAD refused.** A named branch is required.
2. **BASE_REF never silently defaulted.** No hard-coded "main" fallback — wrong base is an invisible failure. Aborts if `git config branch.<n>.gh-merge-base` and `gh repo view` both fail.
3. **PR scope derived from `git merge-base HEAD origin/$BASE_REF` against a freshly fetched remote ref.** Local `$BASE_REF` is never trusted for diff scope — it can be stale.
4. **Behind or diverged from upstream is refused.** Only no-upstream / ahead / in-sync are safe states.
5. **Never `--force` push.**
6. **No co-author trailer in the PR body.** Commits already carry it.

## Pipeline

### Step 0 — Argument parsing

```bash
ARGS="${1:-}"
AUTO_CONFIRM=0
for tok in $ARGS; do
  case "$tok" in
    Y|y) AUTO_CONFIRM=1 ;;
    *)   echo "ERROR: unknown arg '$tok' (only 'Y' is accepted)" >&2; exit 2 ;;
  esac
done
```

### Step 0.5 — Branch sanity

```bash
CURRENT_BRANCH=$(git branch --show-current)
if [ -z "$CURRENT_BRANCH" ]; then
  echo "ERROR: detached HEAD. Check out a named branch first." >&2
  exit 2
fi
```

### Step 1 — Resolve BASE_REF and PR_BASE

```bash
# Per-branch override → repo default → ABORT (no silent fallback to "main").
BASE_REF=$(git config --get "branch.${CURRENT_BRANCH}.gh-merge-base" 2>/dev/null) || \
BASE_REF=$(gh repo view --json defaultBranchRef -q .defaultBranchRef.name 2>/dev/null) || {
  echo "ERROR: could not resolve base branch. Set branch.${CURRENT_BRANCH}.gh-merge-base or check 'gh auth status'." >&2
  exit 2
}

# Need a fresh remote ref for an authoritative merge-base.
git fetch --quiet origin "$BASE_REF" || {
  echo "ERROR: git fetch origin $BASE_REF failed." >&2
  exit 2
}

PR_BASE=$(git merge-base HEAD "origin/$BASE_REF") || {
  echo "ERROR: no merge-base between HEAD and origin/$BASE_REF." >&2
  exit 2
}
```

### Step 2 — Preflight (after PR_BASE known)

```bash
# (a) Refuse to PR base into base.
if [ "$CURRENT_BRANCH" = "$BASE_REF" ]; then
  echo "ERROR: refuse to open a PR from $BASE_REF into itself." >&2
  exit 2
fi

# (b) Refuse dirty working tree.
if ! git diff --quiet || ! git diff --cached --quiet; then
  echo "ERROR: working tree has unstaged or staged changes. Commit or stash first." >&2
  exit 2
fi

# (c) Refuse if no commits ahead of PR_BASE.
COMMITS_AHEAD=$(git rev-list --count "$PR_BASE..HEAD")
if [ "$COMMITS_AHEAD" -eq 0 ]; then
  echo "ERROR: no commits to ship — HEAD is even with origin/$BASE_REF." >&2
  exit 2
fi
```

### Step 3 — Upstream state machine (safe push)

```bash
upstream=$(git rev-parse --abbrev-ref --symbolic-full-name '@{u}' 2>/dev/null || true)

if [ -z "$upstream" ]; then
  # No upstream — set tracking on first push.
  git push -u origin "$CURRENT_BRANCH" || { echo "ERROR: initial push failed." >&2; exit 2; }
else
  LEFT=$(git rev-list --count "${upstream}..HEAD")    # local-only commits
  RIGHT=$(git rev-list --count "HEAD..${upstream}")   # remote-only commits

  if [ "$RIGHT" -gt 0 ]; then
    echo "ERROR: local branch is behind/diverged from $upstream by $RIGHT commit(s)." >&2
    echo "  Pull or rebase before requesting a PR." >&2
    exit 2
  fi

  if [ "$LEFT" -gt 0 ]; then
    git push || { echo "ERROR: push failed." >&2; exit 2; }
  fi
  # If LEFT==0 and RIGHT==0, already in sync — proceed to PR creation.
fi
```

### Step 4 — Autogenerate title and body (using PR_BASE)

```bash
# Title: most recent commit subject, truncated to 70 chars.
RAW_TITLE=$(git log -1 --format='%s')
TITLE=$(printf '%s' "$RAW_TITLE" | cut -c1-70)

# Body: ## Summary (bulleted commit subjects since PR_BASE) + ## Test plan placeholder.
SUMMARY_BULLETS=$(git log "$PR_BASE..HEAD" --reverse --format='- %s')
BODY=$(cat <<EOF
## Summary
$SUMMARY_BULLETS

## Test plan
- [ ] (replace with actual verification steps)
EOF
)
```

### Step 5 — Confirmation prompt (unless `Y`)

```bash
if [ "$AUTO_CONFIRM" -eq 0 ]; then
  cat >&2 <<EOF

About to open PR:
  base:    $BASE_REF
  head:    $CURRENT_BRANCH
  commits: $COMMITS_AHEAD

  ──── title ────
  $TITLE

  ──── body ────
$BODY
  ──────────────

EOF
  printf 'Open PR? [y/N] ' >&2
  read -r reply
  case "$reply" in
    y|Y|yes|YES) ;;
    *) echo "Aborted." >&2; exit 0 ;;
  esac
fi
```

### Step 6 — Create the PR

```bash
PR_URL=$(gh pr create --title "$TITLE" --body "$BODY" --base "$BASE_REF") || {
  echo "ERROR: gh pr create failed." >&2
  exit 2
}
```

### Step 7 — Print PR URL

```bash
echo ""
echo "✓ PR opened:"
echo "  $PR_URL"
echo ""
echo "Next steps:"
echo "  - Review the PR body and edit the Test plan if needed: gh pr edit"
echo "  - When ready to merge: /lifeline:approve-pr"
```

## State machine summary

| Pre-condition | Outcome |
| --- | --- |
| Detached HEAD | ABORT (Step 0.5) |
| Cannot resolve BASE_REF | ABORT (Step 1) |
| Cannot fetch origin/BASE_REF | ABORT (Step 1) |
| Current branch == BASE_REF | ABORT (Step 2a) |
| Working tree dirty | ABORT (Step 2b) |
| No commits ahead of PR_BASE | ABORT (Step 2c) |
| No upstream set | `git push -u`, continue |
| Behind / diverged from upstream | ABORT (Step 3) |
| Ahead of upstream | `git push`, continue |
| In sync with upstream | continue (no push) |
| Confirmation declined | exit 0 (graceful no-op) |
| `gh pr create` failed | ABORT (Step 6) |
| Success | print URL and next-step hints (Step 7) |

## Notes

- **Why `gh repo view --json defaultBranchRef`** instead of `gh pr base default`: the latter is not a valid `gh` subcommand. The `gh pr create` command itself uses `branch.<current>.gh-merge-base` if set, else the repo default branch — this skill mirrors that resolution explicitly so the same value can be used for the preflight commit-count check.
- **Why merge-base against fresh `origin/$BASE_REF`**: the local `$BASE_REF` may have been updated locally without fetching, or the remote may have advanced since the last fetch. The merge-base against the fresh remote ref is what GitHub's PR diff will compute against — this skill matches that computation exactly so the commit count and Summary bullets are accurate.
- **No `--force` push, ever.** If the upstream state machine refuses the push, the user must reconcile manually. Forcing would silently overwrite collaborators' work.
